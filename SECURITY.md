## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

We take the security of this project seriously. If you believe you have found a security vulnerability, please report it using one of the following methods:

1.  **GitHub Private Vulnerability Reporting:** [Optional: Mention if you have this enabled on your repo].
2.  **Email:** Send an email to [Devcode940@gmail.com]. 

### What to include in your report:
To help us resolve the issue quickly, please include:
*   A description of the vulnerability.
*   Steps to reproduce the issue (a proof-of-concept script or screenshots are helpful).
*   Potential impact (e.g., remote code execution, data leak).

### Our Process
*   **Acknowledgment:** You will receive an acknowledgment of your report within [e.g., 48 hours].
*   **Investigation:** We will investigate the issue and may contact you for further information.
*   **Fix:** Once confirmed, we will work on a patch. We aim to resolve critical issues within [e.g., 30 days].
*   **Disclosure:** After a fix is released, we will coordinate a public disclosure.
